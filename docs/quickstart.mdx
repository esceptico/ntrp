## Prerequisites

- Python 3.13+ with [uv](https://docs.astral.sh/uv/)
- [Bun](https://bun.sh) for the desktop app when running from source
- An LLM provider: Anthropic, OpenAI, Google, OpenRouter, custom OpenAI-compatible endpoint, or OpenAI account sign-in

## Install

<Tabs>
  <Tab title="Backend package">
    ```bash
    uv tool install ntrp    # or: pip install ntrp
    ```
  </Tab>
  <Tab title="From source">
    ```bash
    git clone https://github.com/esceptico/ntrp.git
    cd ntrp
    just install
    ```
  </Tab>
</Tabs>

<Note>
The desktop client currently runs from source with `bun run dev` or packaged builds from `apps/desktop`. The old terminal client has been removed.
</Note>

## Set up

<Steps>

### Set your LLM provider (optional)

You can export a provider API key now, or configure a provider interactively in the desktop app after connecting. OpenAI can be used either with an API key or with browser sign-in through the OpenAI Codex provider.

```bash
# Pick one, or skip and configure from the desktop app:
export ANTHROPIC_API_KEY=sk-ant-...
export OPENAI_API_KEY=sk-...
export GEMINI_API_KEY=...
```

If you want to use an OpenAI subscription/account instead of an API key, skip the key export and choose **OpenAI Codex** in provider onboarding. ntrp opens a browser sign-in and stores refreshable OAuth tokens in `~/.ntrp/openai-codex-auth.json`.

### Start the server

<Tabs>
  <Tab title="Installed">
    ```bash
    ntrp-server serve
    ```
  </Tab>
  <Tab title="From source">
    ```bash
    just server
    ```
  </Tab>
</Tabs>

On first run, ntrp generates an API key and prints it:

```
Your API key: nBx7k2...
Enter this in a desktop client to connect. It won't be shown again.
```

<Warning>Copy this key — it's only shown once. Use `ntrp-server serve --reset-key` to generate a new one.</Warning>

### Start the desktop app

In a separate terminal:

```bash
just desktop
```

Or directly:

```bash
cd apps/desktop
bun run dev
```

Paste your API key in the connection screen. If no LLM provider is configured, provider onboarding appears so you can connect a provider and choose defaults.

</Steps>
