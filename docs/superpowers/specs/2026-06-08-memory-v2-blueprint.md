Append to the existing file. `Kind` is an **open set stored as plain `str`** (a `StrEnum` only seeds the four values; an unseen kind from the LLM must not raise — per the no-defensive-code lesson).

```python
class Kind(StrEnum):
    FACT = "fact"
    ACTION = "action"
    PREFERENCE = "preference"
    NOTE = "note"


@dataclass
class Record:
    id: str
    text: str                       # self-contained; survives retrieval alone
    kind: str = Kind.NOTE           # open set — plain str, unseen kinds don't crash
```

### A.2 `memory/records.py` — RecordStore (NEW)

Own connection to `config.memory_db_path` (same db the Curator owns), `vec=False`. Mirrors the Curator's lazy-connect pattern exactly (`curator.py:333-344`). Vector half delegates to the injected `SearchIndex`; FTS half is a local `records_fts`. The two databases (`memory.db` rows ↔ `search.db` vectors) bridge in Python on the record `id`, identical to `_hybrid_search_messages` (`context/store.py:2761-2815`).

**Schema (created on first connect, alongside the Curator's `meta` table):**

```sql
CREATE TABLE IF NOT EXISTS records (
    id TEXT PRIMARY KEY,
    text TEXT NOT NULL,
    kind TEXT NOT NULL DEFAULT 'note',
    scope_kind TEXT NOT NULL,
    scope_key TEXT,
    created_at TEXT NOT NULL,
    last_confirmed_at TEXT NOT NULL,
    superseded_by TEXT,
    pinned INTEGER NOT NULL DEFAULT 0,
    source_ref TEXT            -- JSON (SourceRef.to_dict) or NULL
);
CREATE INDEX IF NOT EXISTS idx_records_scope ON records(scope_kind, scope_key);
CREATE INDEX IF NOT EXISTS idx_records_active ON records(superseded_by);

CREATE VIRTUAL TABLE IF NOT EXISTS records_fts USING fts5(
    text,
    content='records',
    content_rowid='rowid'
);

-- triggers mirror context/store.py:921-941 (_FTS_TRIGGERS), single text column
CREATE TRIGGER IF NOT EXISTS records_ai AFTER INSERT ON records BEGIN
    INSERT INTO records_fts(rowid, text) VALUES (new.rowid, new.text);
END;
CREATE TRIGGER IF NOT EXISTS records_ad AFTER DELETE ON records BEGIN
    INSERT INTO records_fts(records_fts, rowid, text) VALUES ('delete', old.rowid, old.text);
END;
CREATE TRIGGER IF NOT EXISTS records_au AFTER UPDATE ON records BEGIN
    INSERT INTO records_fts(records_fts, rowid, text) VALUES ('delete', old.rowid, old.text);
    INSERT INTO records_fts(rowid, text) VALUES (new.rowid, new.text);
END;
```

> Assumption (recon-stated): `records` has a TEXT PK `id`, so `records_fts` keys on SQLite's implicit `rowid` (an extra INTEGER alias), not on `id`. This matches the transcript FTS which keys on `rowid`. The `rowid`→`id` mapping is recovered when hydrating FTS hits (`SELECT id, ... WHERE rowid IN (...)`).

**Public signatures:**

```python
class RecordOp(StrEnum):            # transient — dreamer LLM output, not persisted
    ADD = "ADD"
    UPDATE = "UPDATE"
    SUPERSEDE = "SUPERSEDE"
    NOOP = "NOOP"


class RecordStore:
    def __init__(self, db_path: Path, search_index: object | None = None) -> None:
        self._db_path = db_path
        self._search_index = search_index   # may be None -> FTS-only
        self._conn = None
        self._conn_lock = asyncio.Lock()

    async def add(self, text: str, *, kind: str = Kind.NOTE, scope: Scope,
                  pinned: bool = False, source_ref: SourceRef | None = None) -> Record: ...
    async def supersede(self, old_id: str, new_id: str) -> None: ...   # sets old.superseded_by = new_id
    async def confirm(self, record_id: str) -> None: ...               # bump last_confirmed_at = now_iso()
    async def update(self, record_id: str, text: str) -> None: ...     # re-text + re-index + confirm
    async def delete(self, record_id: str) -> None: ...                # hard delete + index delete
    async def get(self, record_id: str) -> Record | None: ...
    async def search(self, query: str, *, scope: Scope | None = None,
                     kinds: list[str] | None = None, limit: int = 10,
                     include_superseded: bool = False) -> list[Record]: ...
    async def list(self, scope: Scope, *, pinned_only: bool = False,
                   limit: int = 50) -> list[Record]: ...
    async def close(self) -> None: ...

    async def _ensure_conn(self): ...   # mirrors curator._ensure_conn (curator.py:333-344)
```

**Exact calls each method makes:**

- `add(...)`:
  1. `id = uuid4().hex` (recon assumption — spec only says TEXT PK; uuid is the obvious generator).
  2. `INSERT INTO records (...) VALUES (...)`; `await conn.commit()` (triggers populate `records_fts`).
  3. **Vector index (fire-and-forget, mirror `context/store.py:1275-1284`):**
     ```python
     if self._search_index is not None and text.strip():
         asyncio.create_task(self._search_index.upsert(
             source="record", source_id=record.id,
             title=f"{record.kind} record", content=record.text,
             metadata={"record_id": record.id, "scope_kind": scope.kind.value,
                       "scope_key": scope.key, "kind": record.kind},
         ))
     ```
     `SearchIndex.upsert` (`search/index.py:47-63`) embeds unconditionally — it is **not** gated by `should_embed`/`EMBED_SOURCES` (that gate only guards `sync()`, `index.py:75`). So `source="record"` indexes with **zero change to `EMBED_SOURCES`**, exactly as `"transcript"` works today.
- `update(id, text)`: `UPDATE records SET text=?, last_confirmed_at=? WHERE id=?`; re-run the same `SearchIndex.upsert` (content-hash short-circuit at `index.py:55-58` no-ops on unchanged text).
- `delete(id)`: `DELETE FROM records WHERE id=?` + `await self._search_index.delete("record", id)` (`index.py:65-66`).
- `supersede(old_id, new_id)`: `UPDATE records SET superseded_by=? WHERE id=?` — never hard-deletes; the old record's vector stays indexed but `search()` filters it out via the `superseded_by IS NULL` post-filter.
- `confirm(id)`: `UPDATE records SET last_confirmed_at=? WHERE id=?`.
- `search(...)` — hybrid, clones `_hybrid_search_messages` (`store.py:2761-2815`):
  ```python
  from ntrp.constants import RRF_K
  from ntrp.search.retrieval import rrf_merge
  from ntrp.database import serialize_embedding

  # FTS leg (local records_fts in memory.db), term-OR + quoted, mirror search/store.py:347-391
  fts_ranked = [(rid, 1.0) for rid in fts_hit_ids_in_rank_order]

  # Vector leg (search.db), mirror store.py:2782-2795
  vec_ranked = []
  index = self._search_index
  if index is not None:
      try:
          emb = await index.embedder.embed_one(query)
          raw = await index.store.vector_search(
              serialize_embedding(emb), sources=["record"], limit=max(limit * 4, 40))
          for item_id, score in raw:                       # store.py:308-345
              item = await index.store.get_by_id(item_id)  # store.py:167-189
              meta = item.metadata if item and item.metadata else None
              if not meta or "record_id" not in meta:
                  continue
              vec_ranked.append((meta["record_id"], score))
      except Exception:
          _logger.warning("record vector search failed; FTS-only", exc_info=True)

  fused = rrf_merge([fts_ranked, vec_ranked], k=RRF_K)     # retrieval.py:13-26
  ordered = sorted(fused, key=lambda rid: fused[rid], reverse=True)
  # hydrate rows from memory.db by id, then apply post-filters:
  #   - scope (scope_kind/scope_key) when scope is not None
  #   - kinds when provided
  #   - superseded_by IS NULL unless include_superseded
  # take first `limit` survivors.
  ```
  > Critical (recon-verified): RecordStore must run its **own** scoped vector + FTS legs and fuse with the free `rrf_merge`. It cannot delegate to `SearchIndex.search` (`index.py:113`) — that fuses globally and returns `SearchResult` objects with no way to re-filter by `superseded_by`/scope before paging.
- `list(scope, pinned_only, limit)`: plain `SELECT * FROM records WHERE scope_kind=? AND (scope_key IS ? ) [AND pinned=1] AND superseded_by IS NULL ORDER BY last_confirmed_at DESC LIMIT ?`. No LLM, no vector.

`db_connect` import + usage mirrors the curator: `from ntrp.database import connect as db_connect` (`curator.py:15`), `conn = await db_connect(self._db_path)` (`curator.py:338`, `vec=False`).

### A.3 `memory/lenses.py` — LensStore (NEW)

Same `memory.db`, own connection (or share RecordStore's). Holds the record store + a cheap LLM for the cascade.

```sql
CREATE TABLE IF NOT EXISTS lenses (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL UNIQUE,
    criterion TEXT NOT NULL,
    scope_kind TEXT,            -- NULL = all scopes
    scope_key TEXT,
    created_at TEXT NOT NULL
);
```

```python
@dataclass
class Lens:
    id: str
    name: str
    criterion: str
    scope: Scope | None
    created_at: str

class LensStore:
    def __init__(self, db_path: Path, records: RecordStore,
                 llm=None, model: str | None = None,
                 reasoning_effort: str | None = None) -> None: ...

    async def create(self, name: str, criterion: str, *, scope: Scope | None = None) -> Lens: ...
    async def list(self) -> list[Lens]: ...
    async def get(self, name: str) -> Lens | None: ...
    async def delete(self, name: str) -> bool: ...
    async def view(self, name: str, *, limit: int = 20) -> list[Record]: ...
    async def close(self) -> None: ...
```

`view(name)` (spec §Lenses):
1. `lens = await self.get(name)` → `records = await self._records.search(lens.criterion, scope=lens.scope, limit=limit)`.
2. **Optional cascade LLM filter** on the top-K only when the criterion is *non-topical* (logic/status/negation/behavior). Topical criteria skip the LLM (semantic search is enough). 
   > Assumption (recon-stated, spec leaves the discriminator open): classify "non-topical" with a cheap heuristic — criterion contains negation/comparison/temporal/status tokens (`not`, `without`, `never`, `still`, `overdue`, `before`, `after`, `unfinished`, `blocked`, `>`, `<`). When matched, one LLM call: "Return the ids of records that satisfy: {criterion}. Records:\n{id: text...}". Parse returned ids → filter. When `llm is None`, always skip the cascade (FTS/vector only). Assert call-count in tests to lock this behavior.

**`_SYSTEM_PROMPT` (curator.py:78-93)** — keep the doc rules, add a records section. One call emits both (spec §Rework step 3: "Prefer ONE LLM call emitting both the doc update and the record-ops"). Shape:

```
You maintain (1) a durable MEMORY DOC and (2) a set of atomic RECORDS for the
user (or a project). Given the CURRENT DOC, the EXISTING SIMILAR RECORDS, and
NEW CONVERSATION TURNS, return a single JSON object:

{
  "doc": "<full updated markdown doc>" | "<<NO_CHANGE>>",
  "records": [
    {"op": "ADD",       "text": "<self-contained statement>", "kind": "fact|action|preference|note"},
    {"op": "UPDATE",    "id": "<existing id>", "text": "<corrected statement>"},
    {"op": "SUPERSEDE", "id": "<existing id>", "text": "<replacement statement>", "kind": "..."},
    {"op": "NOOP",      "id": "<existing id>"}
  ]
}

Rules:
(1) NOVELTY GATE — if the turns contain nothing durable AND new, set "doc" to
    "<<NO_CHANGE>>" and "records" to []. Transient task state does not count.
(2) Records are atomic, self-contained (resolve pronouns inline), typed by
    FUNCTION not subject. Choose op against the EXISTING SIMILAR RECORDS:
    ADD (new), UPDATE (edit existing), SUPERSEDE (replace a now-wrong one),
    NOOP (reconfirm an unchanged one). Use ONLY ids from EXISTING SIMILAR RECORDS.
(3) Merge/dedupe the doc; budget ~8000 chars.
(4) Output ONLY the JSON object.
```

> The LLM must **never invent ids** (self-correcting-interfaces lesson). Pre-search and pass the candidate set into the prompt; it picks from real ids only.

**`curate_session` body (curator.py:184-206)** — new control flow:
```python
doc = await self._doc_store.load(scope)
doc_body, ops = await self._complete(scope, doc.content, turns, watermark)

if doc_body is None and not ops:
    return False                                   # LLM failure -> retry next time

# doc path (unchanged): only when a real body came back
if doc_body is not None and not self._is_no_change(doc_body):
    budget = self._doc_store.char_budget
    if len(doc_body) > budget:
        doc_body = await compress_to_budget(self._llm, model=self._model,
            reasoning_effort=self._reasoning_effort, body=doc_body, budget=budget)
    await self._doc_store.write(scope, doc_body)
```

## Assumptions where recon was uncertain
- **`id` generation:** `uuid4().hex` (spec only mandates TEXT PK).
- **`records_fts` rowid:** keyed on SQLite implicit `rowid`, not `id` (TEXT PK can't be an FTS5 external-content rowid). `rowid`→`id` recovered on hydration.
- **Lens cascade discriminator:** heuristic token match (negation/comparison/temporal/status) selects "non-topical"; spec leaves the exact rule open. Locked by call-count assertions in tests.
- **Pinned-record injection plumbing:** via a new `memory_records` field on `ChatDeps` (cleaner than threading through the doc store), matching how `memory_curator` is already a `ChatDeps` field.
